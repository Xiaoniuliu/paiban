## 1. 模块概述

### 1.1 功能简介
系统设置模块提供航空公司排班系统的全局配置功能，包括用户管理、角色权限、基础信息配置、航班运行规则、合规规则、资质要求等核心配置项。

### 1.2 用户角色
| 角色 | 权限 |
|------|------|
| 管理员 (admin) | 全部权限 |
| 排班员 (roster) | 部分配置查看 |
| 机组人员 (crew) | 仅个人设置 |
| 监控员 (monitor) | 只读权限 |

### 1.3 数据来源
参考 `index.html` 中的"系统设置"部分

## 4. 数据结构

### 4.1 用户表 (sys_user)

```sql
CREATE TABLE sys_user (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    username VARCHAR(50) NOT NULL UNIQUE COMMENT '登录账号',
    password VARCHAR(100) NOT NULL COMMENT '加密密码',
    real_name VARCHAR(50) COMMENT '真实姓名',
    phone VARCHAR(20) COMMENT '手机号',
    status TINYINT DEFAULT 1 COMMENT '状态 0-禁用 1-正常',
    create_time DATETIME DEFAULT NOW(),
    update_time DATETIME DEFAULT NOW()
);
```

### 4.2 角色表 (sys_role)

```sql
CREATE TABLE sys_role (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    role_name VARCHAR(50) NOT NULL COMMENT '角色名称',
    role_code VARCHAR(50) UNIQUE COMMENT '角色标识(admin/user/guest)',
    remark VARCHAR(255) COMMENT '备注',
    create_time DATETIME DEFAULT NOW()
);
```

### 4.3 权限表 (sys_permission)

```sql
CREATE TABLE sys_permission (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    perm_name VARCHAR(50) NOT NULL COMMENT '权限名称',
    perm_name_zh VARCHAR(100) COMMENT '中文展示名',
    perm_name_en VARCHAR(100) COMMENT '英文展示名',
    perm_url VARCHAR(255) COMMENT '接口URL/菜单路由',
    perm_code VARCHAR(100) UNIQUE COMMENT '权限标识 sys:user:list',
    perm_type TINYINT COMMENT '权限类型 1-菜单 2-按钮 3-接口',
    parent_id BIGINT DEFAULT 0 COMMENT '父权限ID',
    sort INT DEFAULT 0
);
```

权限表要求：

- `perm_code` 必须使用稳定英文 key，权限判断只使用 key，不使用展示文案。
- `perm_name_zh / perm_name_en` 用于菜单、按钮、权限树展示。
- 菜单权限的父子关系必须与主 FRD 菜单表一致。

### 4.4 用户角色关联表 (sys_user_role)

```sql
CREATE TABLE sys_user_role (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    user_id BIGINT NOT NULL,
    role_id BIGINT NOT NULL,
    UNIQUE KEY uk_user_role (user_id, role_id)
);
```

### 4.5 角色权限关联表 (sys_role_permission)

```sql
CREATE TABLE sys_role_permission (
    id BIGINT PRIMARY KEY AUTO_INCREMENT,
    role_id BIGINT NOT NULL,
    perm_id BIGINT NOT NULL,
    UNIQUE KEY uk_role_perm (role_id, perm_id)
);
```

### 4.6 用户偏好表 (user_preference)

```sql
CREATE TABLE user_preference (
    user_id BIGINT PRIMARY KEY,
    language_code VARCHAR(10) NOT NULL DEFAULT 'zh-CN',
    display_timezone_mode VARCHAR(20) NOT NULL DEFAULT 'UTC_PLUS_8',
    updated_at DATETIME NOT NULL DEFAULT NOW(),
    updated_by BIGINT NULL
);
```

字段要求：

- `language_code` 固定支持 `zh-CN / en-US`。
- `display_timezone_mode` 固定支持 `UTC / UTC_PLUS_8`。
- 登录后用户切换语言或展示时区必须写入该表。
- 后端返回时间仍保持 UTC；前端按该偏好进行展示格式化。

### 4.7 系统配置表 (sys_config)

| 字段 | 类型 | 说明 |
|------|------|------|
| id | bigint | 主键 |
| config_key | varchar(100) | 配置键 |
| config_value | varchar(500) | 配置值 |
| config_type | varchar(20) | 配置类型 |
| is_system | tinyint | 是否系统配置 |

## 6. 业务流程

### 6.1 登录流程

```
用户输入账号密码
    ↓
系统校验账号是否存在、是否被禁用
    ↓
密码加密比对（BCrypt）
    ↓
登录成功 → 查询该用户的所有角色 + 所有权限
    ↓
生成 Token（JWT）返回前端，包含用户 ID、角色、权限列表
```

### 6.2 权限校验流程

```
前端请求接口 → 携带 Token
    ↓
后端拦截器解析 Token → 获取用户权限列表
    ↓
判断当前接口是否在用户权限中
    ↓
有权限放行，无权限返回 403
```

### 6.3 配置修改流程

```
管理员进入系统设置
    ↓
选择配置模块
    ↓
修改配置项
    ↓
点击"保存配置"
    ↓
系统验证必填项
    ↓
保存到数据库
    ↓
显示保存成功提示
```

### 6.4 批量导入流程

```
点击"批量导入"
    ↓
下载Excel模板
    ↓
填写数据
    ↓
上传Excel文件
    ↓
系统校验格式和数据
    ↓
预览导入结果
    ↓
确认导入
    ↓
返回导入成功/失败数量
```

## 7. 异常处理

| 场景 | 处理方式 |
|------|----------|
| 必填项为空 | 红色提示，不允许保存 |
| 数字格式错误 | 提示正确格式 |
| 保存失败 | 显示错误信息，允许重试 |
| 锁定规则被修改 | 记录操作日志，发送告警 |
| 导入文件格式错误 | 提示具体错误行和原因 |
| 导入数据校验失败 | 显示失败行，可跳过或修正 |
| 用户名重复 | 提示"用户名已存在" |
| 删除有关联的角色 | 提示"该角色有关联用户，无法删除" |

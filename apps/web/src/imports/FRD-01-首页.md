## 1. 模块概述

### 1.1 功能简介
首页模块是系统的入口和仪表盘页面，采用Dashboard风格展示整体运行状态。让用户一登录就能快速了解今日整体运行状态，快速发现问题并导航到相关模块处理。

### 1.2 设计风格

| 项目 | 选择 |
|------|------|
| 配色风格 | Apple浅亮风格 |
| 背景色 | #f5f5f7（浅灰） |
| 卡片背景 | #ffffff（纯白） |
| 主色 | #0071e3（Apple蓝） |
| 成功色 | #34c759（翠绿） |
| 警告色 | #ff9500（琥珀橙） |
| 危险色 | #ff3b30（红色） |
| 文字主色 | #1d1d1f |
| 文字次色 | #86868b |
| 图标风格 | 自定义SVG航空图标 |
| 布局风格 | Dashboard仪表盘，左侧侧边栏导航 |

### 1.3 用户角色
| 角色 | 权限 |
|------|------|
| 管理员 (admin) | 查看全部首页数据 |
| 排班员 (roster) | 查看排班相关首页数据 |
| 机组人员 (crew) | 查看个人首页数据 |
| 签派员 (oc) | 查看航班相关首页数据 |
| 监控员 (monitor) | 查看全部首页数据（只读） |

## 2. 功能清单

### 2.1 Header区域

| 功能 | 说明 |
|------|------|
| Logo区域 | 飞机图标 + 系统名称（放大显示） |
| 系统名称 | 中文：全球通货运排班系统 |
| 英文副标题 | GLOBAL CREW SCHEDULING |
| 高度 | 100-120px |

### 2.2 侧边栏导航

本文件来源于旧浅色前端设计包，侧边栏布局、折叠/展开效果、二级菜单视觉层级可作为实现参考；正式菜单文字、路由和业务结构必须以 `FRD_Pilot_Rostering_System_Master.md` 为准，不得照搬旧菜单命名。

| 父菜单 | 子菜单 |
|---------|---------|
| Dashboard / 首页 | direct entry |
| Flight Operations Center / 航班运行中心 | Flight Plan、Operations Data |
| Crew Resources Center / 机组资源中心 | Crew Information、Status Timeline、External Work |
| Rostering Workbench / 排班工作台 | Flight View、Crew View、Unassigned Flights、Validation & Publish、Run-day Adjustments、Post-flight Archive |
| Rule Center / 规则中心 | direct entry |
| Exception Reporting Center / 例外报送中心 | Exception Requests、CDR / AACM |
| Statistics Reports / 统计报表 | direct entry |
| Admin / 系统设置 | Basic Config、Account Management、Role & Permission、Rule Config、Dictionary、Airport & Timezone、Import Mapping、Notification Template、User Preference |
| Pilot Portal / 飞行员端 | My Roster、My Alerts、Status Report、My History、My Preferences |

全局要求：

- 父菜单、子菜单、页面标题、按钮、筛选标签和提示文案必须接入 `zh-CN / en-US` 语言资源。
- 顶部栏固定提供语言切换与 `UTC / UTC+8` 展示时区切换。
- 切换语言或展示时区后，当前页面、侧栏、弹窗、甘特图和报表预览必须同步刷新。

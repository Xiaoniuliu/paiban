# FRD-05：排班工作台

> 使用边界：本文件为旧包/历史导入材料，仅可参考甘特图视觉密度、布局和交互样式；不得作为正式功能需求、菜单、权限、API、数据模型、算法或验收依据。正式业务口径以仓库根目录 `FRD_Pilot_Rostering_System_Master.md` 与 `PHASE1_GANTT_ARCHIVE_WORKFLOW.md` 为准。当前一期甘特图底座固定为 `vis-timeline` 官方 `Timeline` 实例，并通过展示适配层封装；角色口径为 `DISPATCHER / OPS_MANAGER / PILOT / ADMIN`；飞后归档主操作区是 `Post-flight Archive`、归档详情和飞后归档表单，甘特图仅展示状态并提供详情跳转。

## 当前一期新增 P0：甘特图飞后归档

- 甘特图必须标记 `Unarchived / Partially Archived / Archived / Overdue` 航班。
- 航班结束后，排班员点击航班色块打开飞后归档侧栏或弹窗。
- 侧栏展示该航班关联飞行员列表和每名飞行员当前 `CrewArchiveForm` 状态。
- 排班员逐个录入 `actual duty start/end`、`actual fdp start/end`、`flying hour` 或显式勾选无 `FLYING_HOUR`。
- 单个飞行员保存后局部刷新该飞行员状态、航班归档状态和甘特角标。
- 全部关联飞行员完成后，航班自动转为 `Archived`。
- 超过 24 小时未整班归档时，甘特块显示 `Overdue`。
- `DISPATCHER` 可录入和确认；`OPS_MANAGER` 只读；`PILOT` 不进入后台甘特归档录入；`ADMIN` 不参与业务归档。

## 1. 模块概述

### 1.1 功能简介
排班工作台是系统的态势展示模块，当前正式实现基于 `vis-timeline` 官方 `Timeline` 和展示适配层实现航空时间线视图，支持：
- 飞机排班和机组排班的可视化展示
- 手动拖拽分配航班任务给机组
- 一键自动排班（调用 Python OR-Tools 算法服务）
- 时间冲突自动检测（红色标红）
- 分钟级时间精度

### 1.2 用户角色
| 角色 | 权限 |
|------|------|
| 管理员 (admin) | 全部权限，可进行所有排班操作 |
| `DISPATCHER` 排班员 | 核心用户，可进行人工排班、查看甘特图、从甘特图进入飞后归档 |
| `OPS_MANAGER` 运行经理 | 可查看排班、风险和归档状态，一期对甘特飞后归档表单只读 |
| 机组人员 (crew) | 仅查看个人排班 |
| 签派员 (oc) | 查看航班和排班 |
| 监控员 (monitor) | 只读权限 |

### 1.3 数据来源
参考旧包 `index.html` 中的"排班工作台"布局密度和展开效果；正式业务能力、技术底座和角色口径以主 FRD、架构文档和专项流程为准。

| 排班约束 | 自动遵循民航局 AC/OPS/013R00 规则 |

### 2.7 告警级别

| 级别 | 颜色 | 说明 | 甘特图标识 |
|------|------|------|------------|
| 紧急 | 红色 | 阻断性违规，必须立即处理 | 红色闪烁 |
| 警告 | 橙色 | 重要违规，需要关注 | 橙色高亮 |
| 提醒 | 黄色 | 轻微问题，建议处理 | 黄色标记 |
| 提示 | 蓝色 | 提醒信息，可选处理 | 蓝色标记 |

### 2.8 冲突类型

| 冲突类型 | 说明 | 级别 | 规则 |
|----------|------|------|------|
| 时间冲突 | 两个任务时间重叠 | 紧急 | 任务间时间不能重叠 |
| 资质不合规 | 机组无执飞机型资质 | 紧急 | 机组必须持有执飞机型资质 |
| 飞行时长超限 | 超过28天/14天/7天上限 | 紧急 | 28天≤190h，14天≤95h，7天≤55h |
| FDP超限 | FDP超过14小时上限 | 紧急 | FDP≤14h |
| 休息不足 | 任务间休息少于11小时 | 警告 | 最小休息=11h |
| 连续执勤超限 | 连续执勤超过6天 | 警告 | 最多连续执勤6天 |
| 跨时区超限 | 跨时区飞行超过182小时 | 警告 | 28天跨时区≤182h |
| 每月飞行超限 | 月飞行时长超过100小时 | 提醒 | 月≤100h |

## 6. 业务流程

### 6.1 甘特图加载流程

```
用户进入排班工作台页面
    ↓
默认加载当前日期起7天的甘特图数据
    ↓
请求甘特图数据接口
    ↓
查询任务表（schedule_assignment）+ 航班表 + 机组表
    ↓
组装甘特图数据结构（按飞机/机组分组）
    ↓
前端初始化甘特图适配层组件
    ↓
渲染任务块（按类型分色）
    ↓
显示甘特图
```

### 6.2 手动拖拽分配流程

```
用户在甘特图上拖动航班任务块
    ↓
拖动到目标机组行
    ↓
系统校验：
  - 机组是否可用（资质、状态、休息时间）
  - 时间是否冲突
  - 是否违反合规规则
    ↓
如果冲突 → 显示红色警告，弹出冲突详情
    ↓
如果合规 → 确认分配
    ↓
保存分配结果，更新甘特图
```

### 6.3 一键自动排班流程

```
排班员点击"一键排班"
    ↓
弹出排班参数配置（如有）
    ↓
调用 Python 算法服务
  - 传入：航班列表、可用机组信息、约束规则、系统配置
  - 算法：OR-Tools 约束规划求解
  - 返回：最优分配方案 + 冲突报告
    ↓
预览排班结果（甘特图展示）
    ↓
排班员确认或调整
    ↓
确认 → 批量保存分配结果
    ↓
更新各航班状态为"已排班"
```

### 6.4 冲突检测流程

```
系统自动实时检测
    ↓
或排班员点击"合规检查"
    ↓
遍历所有任务分配
    ↓
按民航局规则逐项校验：
  - 时间冲突检测
  - 连续执勤天数检测
  - 休息时长检测
  - FDP上限检测
  - 资质有效期检测
    ↓
生成冲突报告
    ↓
冲突任务块标红
    ↓
显示冲突详情，供排班员处理
```

## 7. 异常处理

| 场景 | 处理方式 |
|------|----------|
| 日期范围超过35天 | 提示"日期范围不能超过35天" |
| 无排班数据 | 显示空白甘特图区域，提示"该日期范围内暂无排班" |
| 数据加载失败 | 显示错误提示，允许重试 |
| Gantt组件加载失败 | 显示错误提示，保留筛选条件，允许重试 |
| 拖拽到不可用机组 | 红色高亮，禁止释放，提示"该机组当前不可用" |
| 时间范围冲突 | 红色高亮，显示冲突详情，提示"与现有任务冲突" |
| 违反合规规则 | 红色高亮，提示具体违规项和规则 |
| 自动排班无可用机组 | 提示"无可用机组满足条件"，列出原因 |
| 算法服务不可用 | 提示"算法服务暂时不可用，请稍后重试" |
| 算法求解超时 | 提示"求解超时，请缩小日期范围" |

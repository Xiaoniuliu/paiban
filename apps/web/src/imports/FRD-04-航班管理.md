## 1. 模块概述

### 1.1 功能简介
航班运行中心负责管理航班计划和航线信息，包括航班增删改查、航线管理、飞机档案管理、航班甘特图可视化。是排班工作台的核心数据来源。

### 1.2 用户角色
| 角色 | 权限 |
|------|------|
| 管理员 (admin) | 全部权限 |
| 排班员 (roster) | 航班和排班相关操作 |
| 机组人员 (crew) | 仅查看个人相关航班 |
| 签派员 (oc) | 航班相关操作 |
| 监控员 (monitor) | 只读权限 |

### 1.3 数据来源
参考 `index.html` 中的"航班运行中心"部分

## 2. 功能清单

### 2.1 航班列表与查询

| 功能 | 说明 |
|------|------|
| 航班分页列表 | 支持分页、排序、筛选 |
| 航班搜索 | 按航班号、航线搜索 |
| 快速筛选 | 按机号、状态（计划中/已排班/已完成/取消）、航线筛选 |

| 筛选字段 | 类型 | 说明 |

| 日期范围 | 日期 | 起降日期范围，必填 |
| 机号 | 下拉 | 全部/AC1/AC2/... |
| 状态 | 下拉 | 全部/计划中/已排班/已完成/取消 |
| 航线 | 下拉 | 全部/澳门→台北/台北→澳门/澳门→首尔/... |

| 状态 | 颜色 | 说明 |
|------|------|------|
| 计划中 | 蓝色 | 航班计划已创建，尚未分配机组 |
| 已排班 | 绿色 | 机组已分配，排班完成 |
| 已完成 | 灰色 | 航班已执行 |
| 取消 | 红色 | 航班已取消 |

### 2.2 航班增删改查

| 功能 | 说明 |
|------|------|
| 新增航班 | 填写航班号、机号、航线、起降时间、执飞/替补机组 |

## 4. 数据结构

### 4.1 航班表 (flight)

| 字段 | 类型 | 说明 |
|------|------|------|
| flight_id | bigint | 主键 |
| flight_no | varchar(20) | 航班号，唯一，如 CGO01 |
| aircraft_id | bigint | 飞机ID，关联 aircraft |
| route_id | bigint | 航线ID，关联 flight_route |
| departure | varchar(50) | 出发地 |
| arrival | varchar(50) | 目的地 |
| dep_time | time | 起飞时间 |
| arr_time | time | 降落时间 |
| flight_duration | int | 飞行时长（分钟） |
| plan_date | date | 计划日期 |
| duty_crew_id | bigint | 执飞机组ID，关联 crew |
| standby_crew_id | bigint | 替补机组ID，关联 crew |
| status | char(1) | 状态：0计划中 1已排班 2已完成 3取消 |
| is_compliant | tinyint | 是否合规：0否 1是 |
| notes | varchar(500) | 备注 |
| create_by | varchar(64) | 创建者 |
| create_time | datetime | 创建时间 |
| update_by | varchar(64) | 更新者 |
| update_time | datetime | 更新时间 |

### 4.2 航线表 (flight_route)

| 字段 | 类型 | 说明 |
|------|------|------|
| route_id | bigint | 主键 |
| route_name | varchar(100) | 航线名称，如"澳门→台北" |
| departure | varchar(50) | 出发地 |

## 6. 业务流程

### 6.1 新增航班流程

```
排班员点击"新增航班"
    ↓
弹出航班表单弹窗
    ↓
填写航班信息（航班号、机号、航线、起降时间）
    ↓
系统校验：
  - 航班号唯一性
  - 起降时间合理性（飞行时长不超过24小时）
  - 机号对应的飞机状态正常
  - 机组未被其他航班占用（冲突检测）
    ↓
校验通过 → 保存航班
    ↓
返回成功提示
    ↓
刷新航班列表
```

### 6.2 新增航线流程

```
管理员点击"新增航线"
    ↓
弹出航线表单弹窗
    ↓
选择出发地和目的地
    ↓
系统自动生成航线名称（出发地→目的地）
    ↓
自动计算时差（目的地时区 - 基地时区）
    ↓
自动判定是否跨时区（时差绝对值 ≥ 6小时）
    ↓
填写飞行时长（分钟）
    ↓
系统校验：
  - 出发地≠目的地
  - 同一航线不存在重复
  - 飞行时长合理（不超过24小时）
    ↓
校验通过 → 保存航线
    ↓
返回成功提示
```

### 6.3 跨时区判定规则

```
选择出发地和目的地后
    ↓
自动计算：
  - 航线名称 = 出发地 + "→" + 目的地
  - 时差 = 目的地时区 - 出发地时区
  - 是否跨时区 = |时差| >= 6
    ↓
例：澳门→洛杉矶
  - 时差 = -8 - (+8) = -16小时
  - 是否跨时区 = |-16| >= 6 → 是
```

## 7. 异常处理

| 场景 | 处理方式 |
|------|----------|
| 航班号重复 | 提示"航班号已存在"，不允许保存 |
| 起降时间跨日超过24小时 | 提示"飞行时长异常（超过24小时）"，不允许保存 |
| 删除已有机组分配的航班 | 提示"该航班已有排班数据，请先取消排班" |
| 机号对应的飞机处于维修/停用状态 | 提示"该飞机当前不可用" |
| 导入Excel格式错误 | 提示具体错误行和原因 |
| 出发地=目的地 | 提示"出发地和目的地不能相同" |
| 航线已存在 | 提示"该航线已存在" |
| 删除已被航班使用的航线 | 提示"该航线已被航班使用，无法删除" |
| 机号重复 | 提示"机号已存在" |
| 删除有航班的飞机 | 提示"该飞机已有航班使用，无法删除" |

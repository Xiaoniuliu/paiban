# 四期范围说明：航班运行中心与机组资源中心资料维护

## 当前结论

三期规则引擎暂缓，等待规则目录人工核对后再继续。当前阶段先补齐每天会用到的运营资料和机组资源台账，不做半自动排班推荐，不做完整法规规则计算。

## 已收敛的四期主线

### 1. 航班运行中心

目标：把航班计划和运行资料从占位/汇总页改成可维护台账。

- `航班计划`：导入批次、批次历史、导入校验入口、字段映射入口、航班池查询。
- `航班池`：支持新增、编辑、取消和状态查看。
- `运行资料`：页内 tabs 管理航班、航线、机场时区、飞机资料。
- 已发布或已归档航班不允许直接改时间或取消，必须走 `排班工作台 > 运行日调整`。

### 2. 机组资源中心

目标：把机组资源中心做成真实主数据和可用性中心。

- `机组信息`：人员档案、资质/执照、小时与限制、执勤日历。
- `状态时间线`：继续维护计划状态块，包含 DUTY / TRAINING / REST / DDO / STANDBY / RECOVERY。
- `外部工作`：维护外部工作、外部飞行、请假、不可用时间和公司外部任务。
- 外部工作保存后应进入机组可用性视图，并可在机组视图甘特中显示为限制/占位块。

### 3. 小时与限制口径

当前只展示已有滚动字段和 actual-only 追踪：

- 连续 7 天 duty。
- 连续 14 天 duty。
- 连续 28 天 duty。
- 连续 28 天 flying。
- 连续 12 个月 flying。
- 单次 FDP 仅在延误、备降、实际归档后按 actual 数据追踪。

FDP/FTL、DDO、Recovery、跨时区恢复、Standby、Positioning、Discretion/CDR/AACM 的完整计算仍归入后续规则引擎阶段。

## 不纳入本轮

- 半自动排班推荐。
- 方案比较优化。
- 高级智能分析。
- 外部系统深度集成。
- 完整规则引擎和法规计算。

## 验收路径

- `/flight-operations/flight-plan` 可查看导入批次和维护航班池。
- `/flight-operations/flights` 可在航班、航线、机场时区、飞机资料四个 tab 查询和维护资料。
- `/crew-status/crew-list` 可在人员档案、资质/执照、小时与限制、执勤日历四个 tab 查看和维护资料。
- `/crew-status/external-work` 不再显示基础人员列表，而是外部工作/不可用时间台账。
- 所有维护动作真实落后端和数据库；删除策略为停用优先。

# 甘特与工作台方向最终纪要

更新时间：2026-04-25

## 1. 最终结论

一期正式确定：甘特图降级为运行态势展示视图，不作为排班事实编辑器，也不作为飞后归档唯一主入口。

产品口径：

- 甘特图负责时间态势展示、快速定位、状态标记、点击查看详情。
- 甘特图不负责拖拽改时间、不负责跨行换机组、不负责直接编辑排班事实。
- 排班、调整、归档、例外、发布、确认均通过任务池、结构化表单、详情抽屉、校验与发布和后端命令完成。
- 后端写模型、规则校验、状态机、版本控制和审计日志是业务事实的唯一安全边界。
- 甘特展示可以降级，但原本想通过甘特实现的业务能力必须在其他模块完整实现。

这条路线优先保证航空安全、规则闭环、审计可追溯和一期可交付性，避免继续把核心产品成败绑定在第三方甘特控件的深度改造上。

## 2. 工作台分工

正式业务操作：

- 任务池承接导入批次、待排航班、待处理冲突、待发布草稿、待归档航班和逾期事项。
- 排班表单承接新建航班任务、修改计划时间、调整机组、填写调整原因。
- 运行日调整表单承接延误、取消、换机组、临时任务、实际运行变更。
- 飞后归档表单承接 actual duty、actual FDP、flying hour、无飞行小时标记和整班归档。
- 校验与发布承接规则命中、阻断项处理、例外/CDR 发起、规则证据查看。
- 详情抽屉承接航班、任务、机组、规则、归档、审计记录的上下文查看和跳转。

甘特图：

- 展示 Flight / Positioning / Standby / Rest / DDO / Recovery / Training / Archive status 等时间块。
- 展示未归档、部分归档、已归档、逾期、规则命中、待处理冲突等状态。
- 支持缩放、滚动、筛选、状态色、角标、tooltip、点击选中。
- 点击色块打开航班/任务/归档/规则详情抽屉，或跳转到对应表单。
- 不直接写入草稿、发布、actual、归档或例外事实。

## 3. 一期甘特能力边界

一期甘特必须支持：

- 50 名飞行员、31 天窗口的横向和纵向浏览。
- 1 / 3 / 7 / 14 / 31 天视图或等价缩放。
- 航班、值勤、休息、培训、归档状态等时间块展示。
- 状态色、角标、tooltip、点击选中。
- 点击色块打开详情抽屉或跳转到对应操作页面。
- `UTC / UTC+8` 展示切换，块位置不变。
- 按角色展示不同入口：排班员可进入操作表单，运行经理只读查看，飞行员只能在飞行员端看本人数据。

一期甘特不承担：

- 直接拖拽改时间。
- 直接跨行拖拽换机组。
- 直接改写正式排班、草稿排班、actual 层或归档层。
- 绕过后端规则校验。
- 批量操作。
- 完整规则处理。
- 自动碰撞避让作为事实源。
- 飞后归档表单录入。

## 4. 复杂能力替代位置

| 能力 | 一期是否需要 | 承接位置 |
|---|---:|---|
| 改计划时间 | 需要 | 排班调整表单 / 运行日调整表单 |
| 换机组 | 需要 | 机组分配表单 / 运行日调整表单 |
| 冲突和碰撞判断 | 需要 | 后端规则引擎 / Rule Center |
| 规则详情 | 需要 | 校验与发布 / 详情抽屉 |
| 归档录入 | 需要 | Post-flight Archive / 归档详情 / 飞后归档表单 |
| 多列冻结、排序、筛选 | 需要 | 任务池、归档队列、规则命中列表 |
| 批量处理 | 需要 | 任务池、Post-flight Archive、Rule Center |
| 报表和导出 | 需要 | Statistics Reports / Data Export |
| 甘特状态展示 | 需要 | 甘特态势视图 |

## 5. 甘特底座选型口径

一期不再为“甘特深度编辑能力”选择控件。选型只服务展示、点击和状态可视化。

正式定版：

- 一期甘特图底座固定为 `vis-timeline` 官方 `Timeline` 实例，参考官方 React 示例实现航空态势时间线，并通过展示适配层封装。
- vis-timeline 负责时间轴、缩放、平移、range item 渲染和点击事件承载。
- React 负责工具栏、左侧资源列、筛选、详情抽屉、权限、i18n 和时区。
- 不再继续评估 SVAR、gantt-task-react 或其他第三方甘特控件作为一期正式底座。
- 不固定自研组件产品名；正式口径使用“vis-timeline 官方 Timeline 甘特态势视图 / 甘特展示适配层”。

vis-timeline 开发硬约束：

- 任何时间块文字、range item 内容和 React 模板改动，必须先参考官方示例：
  [React 16 Components in templates](https://visjs.github.io/vis-timeline/examples/timeline/other/usingReact16.html)
  和官方文档：
  [Timeline documentation](https://visjs.github.io/vis-timeline/docs/timeline/)。
- 色块文字优先使用 vis-timeline 官方 `content` 字段；如需复杂 React 内容，只允许通过官方 `template(item, element)` 或 `visibleFrameTemplate(item, element)` 接入。
- 禁止为了让文字居中、撑宽或裁切而重写 `.vis-item-content`、`.vis-item-overflow`、`transform`、range item 内部宽高和定位模型。
- 禁止在 `content` 内自造 wrapper 后再用 CSS 接管 vis-timeline 的 item 内部布局；这会导致拖动、缩放、窗口裁切时文字与色块脱节。
- 可以改业务映射、颜色 class、tooltip 和外层工具栏；不要改 vis-timeline 官方负责的 range item 内容定位。
- 如果必须覆盖 vis-timeline 内部 CSS，先写明问题、官方方案为何不够、影响范围和回退方式，再做最小覆盖。

不再继续投入：

- `gantt-task-react-main` 仅作为历史 Spike 参考保留，不作为正式底座。
- 为一行多色块、跨行拖拽、pending ghost 做深度改造。
- 以甘特拖拽编辑作为一期验收目标。

## 6. 一期验收口径

一期验收重点从“甘特编辑闭环”调整为“业务闭环 + 甘特态势展示”：

1. 导入或种子航班进入任务池。
2. 排班员通过任务池和排班表单完成机组分配与调整。
3. 后端执行规则校验，阻断项进入校验与发布。
4. 运行经理可审核、查看规则结果、查看发布和归档状态。
5. 航班结束后进入 Post-flight Archive / 归档详情。
6. 排班员逐个完成飞行员归档表单。
7. 单人完成后航班变为部分归档，全员完成后整班已归档。
8. 超过 24 小时未完成时进入逾期状态。
9. 甘特图同步展示航班、机组、规则命中和归档状态，并可点击进入详情。
10. 飞行员端仅能查看本人班表、本人归档摘要和本人相关例外/CDR。

一期收口的详细业务边界见 [PHASE1_ROSTERING_WORKBENCH_CLOSURE.md](</D:/paiban2/PHASE1_ROSTERING_WORKBENCH_CLOSURE.md>)。二期规则接管边界见 [PHASE2_RULE_VALIDATION_CLOSURE.md](</D:/paiban2/PHASE2_RULE_VALIDATION_CLOSURE.md>)。同一机组时间重叠、REST/DDO/FDP/FTL、连续 duty、跨时区 recovery 和完整规则命中池统一放入二期规则校验，不作为甘特图直接编辑能力。

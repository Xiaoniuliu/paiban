# 一期开发顺序：业务闭环优先，甘特态势展示

## Summary

一期不再以“甘特图飞后归档纵切”作为核心开发起点。正式开发顺序调整为：先完成任务池、排班/调整表单、规则校验、飞后归档和审计闭环，再接入甘特态势展示。

甘特图在一期只作为运行态势展示和详情导航视图，不承担拖拽改时间、跨行换机组或直接编辑归档表单。原本期望通过甘特完成的功能，必须由任务池、结构化表单、详情抽屉、校验与发布和后端命令完整实现。

## M0 时间安全护栏

- 保持 UTC 存储/传输，展示时区只影响 UI 文本。
- 前端时间展示必须统一通过时间上下文和时间组件。
- `npm.cmd run check:time` 未通过不得进入业务开发验收。
- 甘特、列表、抽屉、表单、报表预览的时间展示口径必须一致。

## M1 业务写模型与状态机

- 建立航班任务、机组分配、运行日调整、飞后归档、规则结果、例外/CDR、审计日志的核心写模型。
- 新增 `FlightArchiveCase`、`CrewArchiveForm`、归档状态枚举和表单状态枚举。
- 所有写操作使用 `revision / expectedRevision` 防并发覆盖。
- 所有安全相关动作写审计日志。

## M2 任务池与列表入口

- 任务池承接导入批次、待排航班、待处理冲突、待发布草稿、待归档航班、逾期归档。
- 支持筛选、排序、状态标签、批量查看、跳转详情。
- 任务池是排班员的一期主工作入口，不依赖甘特拖拽。

## M3 排班与运行日调整表单

- 排班表单支持创建/调整航班任务、计划时间、机组、备注原因。
- 运行日调整表单支持延误、取消、换机组、临时任务和实际运行变更。
- 保存前展示规则预检结果和时间确认信息。
- 保存后进入后端校验、版本、状态机和审计流程。

## M4 校验与发布

- 展示规则命中、阻断项、警告项、证据快照和处理状态。
- 支持例外/CDR 发起、PIC 决策代录、运行经理审核查看。
- 同一对象命中多条规则时，任务池、详情抽屉、校验与发布和甘特状态展示必须一致。

## M5 飞后归档闭环

- Post-flight Archive / 归档详情 / 飞后归档表单是飞后归档主操作区。
- 航班结束后生成或激活 `FlightArchiveCase`。
- 归档详情展示航班信息、归档截止时间、关联飞行员列表和每人 `CrewArchiveForm` 状态。
- 排班员逐个录入 `actualDutyStartUtc / actualDutyEndUtc / actualFdpStartUtc / actualFdpEndUtc / flyingHour / noFlyingHourFlag`。
- 单人保存后航班变为 `PartiallyArchived` 或保持当前状态。
- 全员完成后自动变为 `Archived`。
- 超过 24 小时未完成变为 `Overdue`，不阻止继续录入，但必须审计。

## M6 权限与飞行员本人边界

- `DISPATCHER` 可进行排班、调整、归档录入和基础合规查看。
- `OPS_MANAGER` 可查看、审核、监督、导出和最终运营确认。
- `PILOT` 只通过 `/api/pilot/me/*` 查看本人班表、提醒、历史、归档摘要、本人相关例外/CDR。
- 后端必须按 `PILOT + sys_user.crew_id` 强制过滤本人数据范围。
- `ADMIN` 仅做系统管理，不参与业务归档录入。

## M7 甘特态势展示接入

- 甘特图底座固定为 `vis-timeline` 官方 `Timeline` 实例，并通过展示适配层封装，不从零自研甘特渲染引擎。
- 甘特读取后端查询投影，只展示，不作为事实源。
- 展示航班、机组、值勤、休息、培训、归档状态、规则命中和逾期提示。
- 支持缩放、滚动、筛选、状态色、角标、tooltip、点击打开详情抽屉或跳转表单。
- 不做拖拽改时间，不做跨行拖拽换机组，不做甘特内归档表单录入。
- Timeline 初始化、items/groups/options 映射和事件细节必须封装在时间线适配层内，不得泄漏到业务页面。

## M8 actual / AACM / 报表接入

- 归档完成后发布 `ArchiveUpdated / ArchiveCompleted` 领域事件。
- 刷新 actual duty、actual FDP、actual violations、累计小时、block time、AACM 待报送清单。
- 统计报表读取归档状态和归档结果摘要，复杂统计后续扩展。

## Test Plan

- 后端：
  - `mvn.cmd test`
  - 覆盖任务池查询、排班/调整保存、规则校验、归档状态机、revision 冲突、审计、角色权限、飞行员本人过滤。
- 前端：
  - `npm.cmd run check:time`
  - `npm.cmd run check:i18n`
  - `npm.cmd run build`
  - `npm.cmd run test:e2e`
- Playwright：
  - 任务池进入排班详情。
  - 表单调整时间和机组后触发规则校验。
  - 校验与发布显示阻断和警告。
  - Post-flight Archive 完成单人归档、部分归档、全员归档、逾期归档。
  - 甘特展示状态并能点击进入详情。
  - 飞行员端只能看到本人数据。
  - `UTC / UTC+8` 切换后展示文本变化，业务状态和审计不被改写。

## Assumptions

- 一期不做自动优化求解。
- 一期不做甘特拖拽编辑。
- 一期不把甘特作为唯一主入口。
- 旧 zip、`apps/web/src/imports` 和失败 Spike 仅作视觉或技术参考，不进入功能需求、菜单、权限、接口或验收范围。

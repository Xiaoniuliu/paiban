# 排班工作台一期收口说明

## 一期主闭环

一期排班工作台收口目标是打通可用业务链路：

`待排航班 -> 保存草稿 -> 校验与发布 -> 发布 -> 发布后运行日调整 -> 飞后归档`

一期已经或必须保留的行为边界：

- 待排航班从任务池或甘特色块进入排班抽屉。
- 排班抽屉支持 `PIC + FO` 必填和 `RELIEF / EXTRA` 额外人员。
- 保存排班只进入草稿，不直接发布。
- 校验与发布页面承担基础发布门槛：待排阻断、PIC/FO 缺失阻断、警告需经理确认。
- 发布后普通排班抽屉只读，变更必须通过运行日调整形成留痕。
- 运行日调整一期支持延误、取消、待命呼出、换人记录和插入休息入口。
- 飞后归档由系统按“已正式排班、有机组、航班已结束”自动进入队列。
- 甘特图只负责态势展示、定位和打开详情，不直接编辑业务事实。

## 一期不做完整规则引擎

以下能力统一放入二期规则校验能力，不作为一期收口阻断：

- 同一机组时间重叠冲突。
- 航班与 `REST / DDO / TRAINING / STANDBY / DUTY / RECOVERY` 等状态块冲突。
- 运行日调整后的连锁冲突。
- FDP / FTL / LNP / WOCL / sector / preceding rest 合规计算。
- REST / DDO / Recovery 合规判断。
- 连续 duty 天数、14 天 DDO、28 天累计等滚动窗口规则。
- 完整规则命中池、证据窗口、规则版本、例外/CDR/AACM 联动。

一期页面可以展示“规则校验待接入 / 二期规则接管”的说明，但不得把这些规则判断伪装成已完成。

## 发布后变更口径

发布后航班不能通过普通排班保存直接覆盖事实：

- 延误：通过运行日调整确认后更新任务计划时间和关联时间轴块。
- 取消：通过运行日调整确认后将航班标记为取消，并从发布/飞后归档队列排除。
- 插入休息：通过运行日调整确认后生成 `REST` 计划状态块。
- 换人：一期先记录运行日调整事项，复杂替换和后续规则重算由二期规则/调整闭环接管。

## 一期验收路径

- 从“待排航班”打开排班抽屉，保存 `PIC / FO` 草稿。
- 在航班视图和机组视图看到对应时间块。
- 在“校验与发布”提交校验，阻断项可回到对应任务处理。
- 校验通过后发布，已发布任务在普通排班抽屉只读。
- 在“运行日调整”创建并确认应用延误、取消或插入休息。
- 航班结束后符合条件的航班自动进入“飞后归档”。
- 归档保存后列表、统计卡片和甘特归档状态刷新。

# 二期规则校验接管说明

## 二期主线

二期主线是让规则校验接管排班、运行日调整和发布门槛，不再只依赖任务状态做基础判断。

一期已完成：

- 待排航班保存草稿。
- 校验与发布基础门槛。
- 发布后普通排班锁定。
- 运行日调整基础应用。
- 飞后归档自动入队。

二期在这些基础上增强，不推倒重做。

## 规则命中池

系统以 `rule_catalog` 和 `violation_hit` 作为规则命中来源。

二期第一批规则：

- 同一机组时间重叠。
- 航班与 `REST / DDO / TRAINING / STANDBY / DUTY / RECOVERY` 状态块冲突。
- PIC / FO / 额外人员基础发布校验。
- 运行日调整应用后的连锁冲突重算。

后续批次规则：

- FDP / FTL / preceding rest。
- DDO 34 小时和 local nights。
- 连续 duty。
- 14 天 DDO。
- 28 天累计。
- Recovery。

## 页面承接

- `校验与发布` 读取规则命中池生成 BLOCK / WARNING 摘要。
- BLOCK / NON_COMPLIANT 阻断发布。
- WARNING / NEEDS_REVIEW 进入经理确认或例外入口。
- `规则中心` 展示 Rule ID、版本、FOM 来源、命中次数和最近命中。
- 甘特图继续只做展示、定位、打开详情，不直接编辑业务事实。

## 运行日调整增强

运行日调整一期已经支持延误、取消、插入休息和应用动作。二期增强为：

- 延误、取消、插入休息应用后触发规则重算。
- 换人补齐原人员、新人员、角色和生效时间窗。
- 换人确认后更新对应 `timeline_block`。
- 所有调整继续保留审计和事件，不做无痕覆盖。

## 非二期范围

以下内容不进入二期主工程：

- 半自动排班推荐。
- 方案比较。
- 高级分析。
- 外部系统集成。
- 移动端深度审批增强。

这些能力放到三期或四期。
